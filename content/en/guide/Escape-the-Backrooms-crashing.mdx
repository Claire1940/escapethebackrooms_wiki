<Section icon={Map} title="Resolving Audio and VR Compatibility Issues">

Even with a high-end PC, specific software conflicts can cause **Escape the Backrooms crashing** or functional errors like missing audio. A documented issue involves the game incorrectly triggering SteamVR even when playing in flat-screen mode. This consumes unnecessary system resources and can lead to a "Low Memory" crash.

**Known Issues and Fixes:**
- **No Audio Playback**: This can occur regardless of output settings. Ensure your default playback device in Windows is set to 48000Hz (DVD Quality) in the Sound Control Panel.
- **Ghost SteamVR Launch**: VR is currently disabled by developers due to bugs, but the game may still attempt to render to a headset. Disconnect any VR hardware before launching the game.
- **PhysX Errors**: Ensure you have the latest NVIDIA PhysX middleware installed, even if you are using an AMD card, as the game uses this for physics calculations.

| **Audio** | Buggy | Reset output to 48kHz / Update drivers |

| **Multiplayer** | Stable | Ensure firewall allows PhysX ports |
